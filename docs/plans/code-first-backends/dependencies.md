    - `serial` replaced with `GENERATED BY DEFAULT AS IDENTITY` on PG

    - `insert_many` overhaul — `InsertMany` helper struct, `last_insert_id` is `Option<Value>`
    - `execute`/`query_one`/`query_all` take SeaQuery statements; raw SQL uses `execute_raw`/`query_one_raw`
    - Full migration guide: https://www.sea-ql.org/blog/2026-01-12-sea-orm-2.0/
    - Entity structure docs: https://www.sea-ql.org/SeaORM/docs/generate-entity/entity-structure/

- sea-orm-migration@2.0.0-rc.37 (latest: 2.0.0-rc.37) — only needed in `yauth-migration` crate for `cargo yauth generate --orm seaorm` scaffold output. Not a runtime dep of the backend. https://crates.io/crates/sea-orm-migration

- toasty@0.3.0 (latest: 0.3.0) — async ORM for Toasty backends. https://crates.io/crates/toasty
  - MSRV 1.94 — current toolchain is 1.93. **Toasty backends require Rust 1.94+.** This blocks implementation until the toolchain is upgraded.
  - Pre-1.0, experimental — expect frequent breaking changes across 0.x releases
  - Schema: `#[derive(toasty::Model)]` derive macros (NOT `.toml` files — that was the old approach)
  - Model attributes: `#[key]`, `#[auto]` (UUID v7 default for Uuid, increment for ints), `#[unique]`, `#[table = "name"]`, `#[has_many]`, `#[belongs_to(key = x, references = y)]`, `#[has_one]`
  - Query API: `Model::get_by_id()`, `Model::all()`, `Model::filter_by_*()`, `Model::filter()`, terminals `.exec()`, `.first()`, `.get()`
  - Create: `toasty::create!(Model { ... }).exec(&mut db).await?` or `Model::create().field(val).exec(&mut db).await?`
  - Connection: `toasty::Db::builder().models(toasty::models!(M1, M2)).connect("postgresql://...").await?`
  - Table prefix: `Db::builder().table_name_prefix("yauth_")` — native support for the yauth prefix
  - Schema mgmt: `db.push_schema()` for dev (creates all tables), `toasty-cli migration generate/apply` for prod
  - Transactions: `db.transaction().await?` with auto-rollback on drop, nested savepoint support
  - Supported DBs: PostgreSQL, MySQL, SQLite, DynamoDB
  - Toasty guide (sparse but primary docs source): https://github.com/tokio-rs/toasty/tree/main/docs/guide/src
  - API docs: https://docs.rs/toasty/0.3.0/toasty/

- Toasty drivers are NOT direct deps — pulled in via toasty feature flags:
  - `toasty/postgresql` enables toasty-driver-postgresql@0.3.0
  - `toasty/mysql` enables toasty-driver-mysql@0.3.0
  - `toasty/sqlite` enables toasty-driver-sqlite@0.3.0

## Approach Decisions
- **SeaORM version**: Use 2.0.0-rc.37 (not 1.x) because 2.0 is the active development line, has the new entity format, and yauth is adding fresh support with no migration burden. Pin exact RC version; bump on stable release.
- **SeaORM entity format**: Use the new SeaORM 2.0 entity format for generated entities — it's more concise and is the recommended format going forward.
- **Toasty model pattern**: Use `#[derive(toasty::Model)]` with Toasty's native `table_name_prefix("yauth_")` builder option. Models live in the backend module, not in `yauth-entity`.
- **Shared entity/model modules**: SeaORM entities are database-agnostic, so all three SeaORM backends share definitions via `backends/seaorm_common/entities/`. Similarly, Toasty models are database-agnostic, shared via `backends/toasty_common/models/`. Each per-dialect backend re-exports the shared module and only differs in pool/connection setup and schema validation.
- **Toasty driver features**: Use toasty's built-in feature flags (`postgresql`, `mysql`, `sqlite`) rather than depending on driver crates directly — toasty re-exports them.
