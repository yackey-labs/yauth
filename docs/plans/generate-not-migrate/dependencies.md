# Dependencies

## Existing (no changes)

- `yauth-migration`@0.8.12 (latest: 0.8.12) — Schema types, DDL generators, plugin schemas. Stays as-is but becomes a build-time-only dependency of `cargo-yauth`. Removed from `yauth` crate's runtime deps.
- `diesel-async`@0.8.0 (latest: 0.8.0) — Async diesel with deadpool pooling. Backend constructors already have `from_pool()` — just remove `new(url)` variants.
- `sqlx`@0.8.x (latest: 0.9.0-alpha.1) — Stay on 0.8.x. 0.9 is alpha. Backends already have `from_pool()`.
- `sea-orm`@2.0.0-rc.37 (latest: 2.0.0-rc.37) — Stay pinned. Backends already have `from_connection()`.
- `clap`@4.6.0 (latest: 4.6.0) — CLI argument parsing for cargo-yauth.
- `dialoguer`@0.11.x (latest: 0.12.0) — Interactive prompts. Could bump but not required for this work.
- `toasty`@0.3.0 — Used in yauth-toasty-example. No changes to the dependency itself.

## To Add

None. This work is a removal/simplification — no new dependencies.

## Approach Decisions

- **Remove `new(url)` constructors vs deprecate**: Remove outright. This is a major version bump anyway (`migrate()` removal is breaking). No deprecation shim.
- **SQL files in diesel_migrations/**: Delete them. `cargo-yauth` already generates SQL from `TableDef` via `yauth-migration`'s DDL generators — these pre-written SQL files are redundant. They were only needed for `include_str!()` in the runtime migration system, which no longer exists.
- **sqlx query files**: Generated by `cargo-yauth`, not shipped in the `yauth` crate. The sqlx backends will use `sqlx::query()` with string literals (already do), not `query_file!()` at the library level. `query_file!()` is for the **user's** project — the generated `.sql` files live in the user's repo.
